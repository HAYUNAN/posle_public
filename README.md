# 🐾 포슬 — Pet Care B2B SaaS

> 애견미용샵 및 펫케어 비즈니스를 위한 올인원 B2B SaaS 플랫폼
>
> 🌐 [posle.kr](https://posle.kr) · 📱 [app.posle.kr](https://app.posle.kr)

기획 · DB 설계 · UI/UX · 프론트엔드 · 백엔드 · 모바일 앱 · 인프라 · 운영 도구까지
전 과정을 **1인 풀스택**으로 설계·구현·배포한 상용 SaaS 프로젝트입니다.

---

## 왜 이 프로젝트를 만들었나

국내 애견미용샵은 약 3~5만 개이며, 그 중 절반 이상이 1인샵입니다.
대부분 수기 예약장이나 카카오톡으로 운영하고 있고, 재방문 관리·고객 데이터·정산을 체계적으로 하는 곳은 거의 없습니다.

포슬은 이 시장의 디지털 전환을 목표로, 예약 관리를 넘어 보호자 앱·케어 모듈·정기결제·초대 보상·메시지 크레딧까지 미용샵 운영에 필요한 모든 것을 하나의 플랫폼으로 제공합니다.

## 아키텍처 하이라이트

### 1. 멀티테넌트 보안 (Row Level Security)
모든 테이블에 Supabase RLS 정책 적용. `get_my_shop_id()` SECURITY DEFINER 함수로 쿼리 레벨에서 샵별 데이터 격리를 강제. 코드에서 `WHERE shop_id = ?`를 빠뜨려도 다른 샵 데이터에 접근 불가.

### 2. 메시지 크레딧 + 푸시 우선 알림
발송 우선순위: Expo Push(무료) → 알림톡(8원) → SMS(20원). 잔액 부족 시 유료 발송 자동 차단. 빌링키 기반 자동충전으로 끊김 없는 서비스. 모든 발송 건별 로그 기록.

### 3. 결제 정산 자동화 + 3자 보호
시술 완료 시 서비스권 → 쿠폰 → 포인트 순서로 자동 차감. 모든 변동은 `benefit_logs`에 기록 + 고객 푸시 알림 + HQ 감사 로그. 차감 취소 시 전부 복원.

### 4. 멀티샵 아키텍처
한 `user_id`에 여러 `staff` 레코드 → 로그인 시 `selected_shop_id` 쿠키로 샵 선택 → 레이아웃에서 쿠키 기반 데이터 분기. 각 샵은 독립 구독·독립 데이터.

### 5. 결제 모니터링 파이프라인
정기결제 성공/실패/일일 요약 → Discord #hq-alerts 실시간 알림. 3회 재시도 후 자동 다운그레이드. HQ에서 건별 토스 API 환불. 예약금 결제 실패도 별도 알림.

### 6. 전자 동의서 퀵 패스
예약 확정 시 1회용 `public_token` 생성 → SMS 자동 발송. 보호자는 앱 설치 없이 웹 브라우저에서 서명 완료.

### 7. 공공데이터 + Google Places 데이터 파이프라인

전국 애견미용샵 데이터를 수집·정제·보강하는 5단계 파이프라인을 직접 설계·구현했습니다.

**Step 1 — 공공데이터 CSV Import**
data.go.kr "동물미용업" 데이터(15,524개) 다운로드 → 영업중 매장만 필터링(10,932개) → EUC-KR 인코딩 처리 → TM 중부원점(EPSG:2097) 좌표를 WGS84(위경도)로 변환하는 수학 공식을 proj4 라이브러리 없이 직접 구현 → Supabase `public_shops` 테이블에 배치 insert.

**Step 2 — 카카오 지오코딩 (좌표 보정)**
좌표가 없는 395개 매장에 대해 카카오 주소검색 API로 주소→좌표 변환 시도. 공공데이터 특성상 상세번지가 `***`로 마스킹되어 있어 71개만 성공. 나머지는 주소 자체가 불완전하여 변환 불가.

**Step 3 — Google Places API 연동 (영업시간·사진·평점)**
Google Places API (New)의 Text Search + Place Details + Place Photo API를 조합하여 매장별 영업시간, 사진(최대 10장), 평점, 리뷰 수, Google Maps URL을 수집. 매장명+주소 기반 매칭 시 이름 유사도 검증(Levenshtein 거리 + 정규화)으로 오매칭 방지. 폐업 확인된 매장은 자동 삭제. 월 $200 무료 크레딧 내에서 운영하기 위해 API 호출별 비용 추적 + $190 도달 시 자동 중단하는 안전장치 구현.

**Step 4 — Google Places 신규 매장 발굴**
전국 466개 시군구를 순회하며 "애견미용/반려동물미용/펫미용" 키워드로 Google Places Text Search → 기존 `google_place_id` 기준 중복 체크 → 신규 매장만 Details + Photo 수집 후 DB insert. 3,040개 신규 매장 발굴. `--resume` 옵션으로 중단 지점부터 이어서 실행 가능.

**Step 5 — 주소 정제**
영어 주소(Google 원본)를 카카오 역지오코딩으로 한국어 도로명 주소로 변환하면서, 원본의 층/호 정보(3F, 101호 등)를 정규식으로 추출하여 뒤에 붙이는 방식으로 상세 주소 보존. 공공데이터의 `***` 마스킹 주소도 좌표 기반 역지오코딩으로 실제 주소로 복원. 지번 주소는 도로명 주소 기반 카카오 주소검색 API로 변환하여 동호수까지 확보.

**결과**: 최종 12,000+ 매장 데이터 확보. 사진 보유 매장 약 2,000개, 영업시간 보유 약 2,000개, 평점 보유 약 2,000개.

## 개발 과정에서 고민한 것들

- **횟수권 vs 월 구독권**: 횟수권은 고액 선결제 후 폐업 리스크 → 월 구독권(포인트+무료서비스 조합, 상한 20만원)
- **SMS 비용 최적화**: 푸시 우선 정책 + 메시지 크레딧 충전제로 원장에게 비용 전가 없이 투명한 과금
- **영업시간 외 문의**: silent(알림만 끔) / block(자동 안내+차단) 모드로 사장도 쉴 수 있게
- **무게 구간 표준화**: "소형견"이 몇 kg인지는 샵마다 다르므로 `{label, max}` 구조로 원장이 직접 설정
- **감사 추적**: 포인트/쿠폰 수동 조작 시 사유 필수 + 고객 알림 + HQ 로그로 3자 보호
- **멀티샵**: 파트타임 직원·다점포 원장 모두 지원. 쿠키 기반 샵 전환으로 UX 단순화
- **공공데이터 좌표 변환**: data.go.kr CSV의 좌표가 TM 중부원점(미터 단위)이라 WGS84 변환 필요. proj4 라이브러리 의존성 문제로 Gauss-Krüger 역변환 공식을 직접 구현
- **Google Places 비용 관리**: 매월 $200 무료 크레딧 초과 방지를 위해 API 호출별 비용 추적 카운터 + $190 도달 시 process.exit 안전장치. 무료 체험판에서는 일반 계정 활성화 안 하면 자동 차단되는 구조 활용
- **주소 마스킹 문제**: 공공데이터 상세번지가 `***`로 마스킹 → 카카오 역지오코딩(좌표→주소)으로 복원. 영어 주소(Google 원본)는 역지오코딩 + 원본 층/호 정보 정규식 추출로 한국어 변환
- **데이터 정합성**: 잘못된 삭제 쿼리로 12,000건 데이터 소실 사고 → CSV 백업에서 전량 복구. 이후 삭제 로직 제거하고 업데이트만 수행하는 방식으로 전환
- **코드 품질 리팩토링**: HQ server actions 1,264줄을 7개 도메인별 파일로 분리. 랜딩 페이지 427줄을 15개 섹션 컴포넌트로 분리. 좌표 변환 중복 코드 공통 유틸 추출. 공통 입력 검증 유틸(`validate.ts`) 도입으로 UUID/이메일/비밀번호 형식 검증. `any` 타입 제거 및 타입 안전성 강화
