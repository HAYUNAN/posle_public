# 🐾 포슬(Posle) — Pet Care B2B SaaS

> 애견미용샵 및 펫케어 비즈니스의 디지털 전환을 위한 올인원 B2B SaaS 플랫폼

기획 · DB 설계 · UI/UX · 프론트엔드 · 백엔드 · 모바일 앱 · 인프라 · 운영 도구까지
전 과정을 **1인 풀스택**으로 설계·구현한 상용 수준의 SaaS 프로젝트입니다.

---

## 왜 이 프로젝트를 만들었나

국내 애견미용샵은 약 3~5만 개(등록 기준 10,172개)이며, 그 중 50~60%가 1인샵입니다.
대부분 수기 예약장이나 카카오톡으로 운영하고 있고, 재방문 관리·고객 데이터·정산을 체계적으로 하는 곳은 거의 없습니다.

포슬은 이 시장의 디지털 전환을 목표로, 예약 관리를 넘어 보호자 앱·케어 모듈·정기결제·초대 보상까지 미용샵 운영에 필요한 모든 것을 하나의 플랫폼으로 제공합니다.

## 아키텍처 하이라이트

### 1. 멀티테넌트 보안 (Row Level Security)
모든 테이블에 Supabase RLS 정책을 적용합니다. `get_my_shop_id()` SECURITY DEFINER 함수로 쿼리 레벨에서 샵별 데이터 격리를 강제하며, 코드에서 `WHERE shop_id = ?`를 빠뜨려도 다른 샵 데이터에 접근할 수 없습니다.

### 2. 푸시 우선 알림 + 방해금지
보호자에게 알림 발송 시 Expo Push 토큰이 있으면 푸시, 없으면 SMS로 자동 폴백합니다. 원장은 방해금지 시간(silent/block)을 설정할 수 있으며, block 모드에서는 고객에게 자동 안내 메시지가 발송됩니다.

### 3. 결제 정산 자동화 + 3자 보호
시술 완료 시 서비스권 → 쿠폰 → 포인트 순서로 자동 차감합니다. 모든 포인트/쿠폰 변동은 `benefit_logs` 테이블에 기록되며, 고객 푸시 알림 + HQ 감사 로그가 동시에 생성됩니다. 차감 취소 시 전부 복원됩니다.

### 4. 예약 겹침 시각화
Greedy Column Assignment 알고리즘으로 동일 시간대 겹치는 예약을 동적 열 인덱스로 분배합니다. SSR(UTC)↔브라우저(KST) 시간대 불일치는 `mounted` 훅으로 방지했습니다.

### 5. 전자 동의서 퀵 패스
예약 확정 시 1회용 `public_token`을 생성해 SMS로 자동 발송합니다. 보호자는 앱 설치 없이 웹 브라우저에서 서명을 완료할 수 있습니다.

### 6. 초대 보상 자동화
빌링 Cron에서 첫 유료 결제를 감지하면 referral 타입에 따라 쿠폰 자동 발급 + 구독료 차감을 처리합니다. 고객→사장, 사장→사장 두 가지 경로를 지원합니다.

## 개발 과정에서 고민한 것들

- **횟수권 vs 월 구독권**: 횟수권은 고액 선결제 후 폐업 리스크가 있어 월 구독권(포인트+무료서비스 조합, 상한 20만원)으로 설계
- **SMS 비용 최적화**: 푸시 우선 정책으로 앱 설치 유도 + SMS는 앱 미설치자만 발송
- **영업시간 외 문의**: 사장도 쉬어야 하니까 — silent(알림만 끔) / block(자동 안내+차단) 모드
- **무게 구간 표준화**: "소형견"이 몇 kg인지는 샵마다 다르므로 `{label, max}` 구조로 원장이 직접 설정. 모든 서비스는 구간별 가격 필수 적용
- **감사 추적**: 포인트/쿠폰 수동 조작 시 사유 필수 + 고객 알림 + HQ 로그로 3자 보호
